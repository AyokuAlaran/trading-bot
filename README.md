# Polymarket Trading Bot

A Claude-powered prediction market trading bot with simulated execution,
P&L tracking, bankroll compounding, and a mobile-friendly browser dashboard.

---

## Project Structure

```
trading-bot/
├── bot.py              # Claude AI agent — analyzes markets, finds edges
├── executor.py         # Trade manager — simulates execution, tracks P&L
├── market_fetcher.py   # Live market loader — Polymarket Gamma API + cache
├── dashboard.html      # Browser dashboard — bankroll, positions, chart
├── dashboard_data.js   # Auto-generated by executor.py (do not edit)
├── mock_markets.json   # 10 sample markets used when USE_LIVE_MARKETS=false
├── markets_cache.json  # Auto-created: last successful live fetch (cache)
├── trades.json         # Persistent trade history (auto-created on first run)
├── requirements.txt    # Python dependencies
├── .env.example        # Template — copy to .env and fill in your key
├── vercel.json         # Vercel deployment config
└── .vercelignore       # Files excluded from Vercel deployment
```

---

## Quick Start

### 1. Install Python dependencies

```bash
cd desktop/trading-bot
pip install -r requirements.txt
```

### 2. Set your Anthropic API key

Copy the template and fill in your key:

```bash
cp .env.example .env        # Mac/Linux
copy .env.example .env      # Windows
```

Edit `.env`:
```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Get your key at: https://console.anthropic.com/settings/keys

### 3. Run the bot

```bash
python bot.py
```

By default the bot analyses the 10 markets in `mock_markets.json`.
To use **live Polymarket markets**, see the [Live Market Data](#live-market-data) section below.

### 4. Open the dashboard

Double-click `dashboard.html` to open it in your browser.
It auto-refreshes every 30 seconds.

**On your phone (same Wi-Fi network):**

```bash
python -m http.server 8080
```

Then visit `http://<your-computer-ip>:8080/dashboard.html` on your phone.
Find your IP with `ipconfig` (Windows) or `ifconfig` (Mac/Linux).

### 5. Run the bot again

Each subsequent run:
- Resolves some open positions (simulated outcomes based on Claude's probability estimates)
- Analyzes the markets again for new opportunities
- Places new trades if edges are still above 5%

After 3–4 runs you'll see a full P&L history in the dashboard.

## Live Market Data

By default the bot reads `mock_markets.json` (offline, no external calls).
To switch to real-time Polymarket data, set `USE_LIVE_MARKETS=true` in your `.env`:

```
ANTHROPIC_API_KEY=sk-ant-your-key-here
USE_LIVE_MARKETS=true
```

### What the live fetcher does

`market_fetcher.py` calls the **Polymarket Gamma API** (`gamma-api.polymarket.com/markets`)
and applies the following filters before returning markets to Claude:

| Filter | Value |
|--------|-------|
| Market status | `active=true`, `closed=false`, `archived=false` |
| Type | Binary only (YES / NO), negRisk markets excluded |
| Minimum 24h volume | $1 000 |
| Minimum liquidity | $1 000 |
| Max markets returned | 20 (highest volume first) |

Markets are mapped to the same schema as `mock_markets.json`, so `bot.py`
and `executor.py` work without any changes.

### Cache & fallback

Every successful fetch is saved to `markets_cache.json`.
If the API is unreachable (timeout, rate-limit, no internet), the bot
automatically falls back to the cache. If the cache is also unavailable,
it falls back to `mock_markets.json`.

Fallback priority:
```
Polymarket Gamma API → markets_cache.json (fresh) → markets_cache.json (stale) → mock_markets.json
```

## Vercel Deployment

The dashboard can be deployed to **Vercel** so you can view it from anywhere —
your phone, a friend's computer, or any browser without running a local server.

### Architecture

The Python bot runs **locally** (or on any server). It writes `dashboard_data.js`
after each run. You push that file to Vercel and the hosted dashboard reads it.

```
[Local machine]                    [Vercel]
  python bot.py
      │
      └─ writes dashboard_data.js ──► vercel deploy ──► https://your-app.vercel.app
```

Only two files are deployed: `dashboard.html` and `dashboard_data.js`.
All Python files, secrets, and raw data are excluded via `.vercelignore`.

### One-time setup

**1. Install the Vercel CLI**

```bash
npm install -g vercel
```

**2. Link the project**

Run this once from the `trading-bot/` directory:

```bash
vercel link
```

Follow the prompts to log in and connect to a Vercel project.

### Updating the dashboard after each bot run

After running `python bot.py`, deploy the updated `dashboard_data.js`:

```bash
vercel --prod
```

Your live dashboard at `https://your-app.vercel.app` will update within seconds.

### Automated deployment with GitHub Actions (optional)

If you push the project to a GitHub repository, you can auto-deploy whenever
the bot writes new data. Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy Dashboard

on:
  push:
    paths:
      - dashboard_data.js

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Vercel
        run: npx vercel --prod --token ${{ secrets.VERCEL_TOKEN }}
```

Add your `VERCEL_TOKEN` (from vercel.com/account/tokens) as a GitHub secret.

With this in place, your workflow becomes:

```bash
python bot.py                  # run the bot locally
git add dashboard_data.js
git commit -m "update dashboard data"
git push                       # triggers auto-deploy
```

### What gets deployed

| File | Deployed? | Why |
|------|-----------|-----|
| `dashboard.html` | ✅ | The UI |
| `dashboard_data.js` | ✅ | Live trade data |
| `*.py` | ❌ | Source code, not needed |
| `trades.json` | ❌ | Raw data (use dashboard_data.js instead) |
| `mock_markets.json` | ❌ | Local only |
| `.env` | ❌ | Secrets — never deployed |

### Dashboard cache headers

`vercel.json` sets:
- `dashboard.html` → no-cache (always fresh)
- `dashboard_data.js` → 30 s cache, stale-while-revalidate (fast loads, near-live data)

## How the Bot Works

### `bot.py` — Claude AI Agent

1. Loads markets from live API or `mock_markets.json` (based on `USE_LIVE_MARKETS`)
2. Sends all markets to Claude (`claude-sonnet-4-6`) with a prompt asking it to:
   - Estimate the TRUE probability of each event
   - Compare against the market price (the YES token price)
   - Calculate the edge: `|true_probability - market_price|`
   - Recommend YES / NO / SKIP for each market
3. Filters to opportunities with edge > 5%
4. Passes each opportunity to `executor.py`

### `market_fetcher.py` — Live Market Loader

Fetches from the Polymarket Gamma REST API when `USE_LIVE_MARKETS=true`.
Filters, maps, and caches markets. Falls back gracefully on any API failure.

### `executor.py` — Trade Manager

- **Bankroll:** Starts at $100
- **Max bet:** 10% of current bankroll per trade (hard cap)
- **Sizing:** Uses Claude's `suggested_bet_fraction` (Kelly-lite) capped at 10%
- **Resolution simulation:** Each run, open positions have a ~40% chance to resolve.
  Outcomes are drawn probabilistically from Claude's estimated true probability,
  so better-analysed trades win more often over time
- **Persistence:** All state is saved to `trades.json` between runs

### `dashboard.html` — Monitoring Dashboard

- **Bankroll card** — Current cash + amount in open positions
- **P&L + ROI** — Net profit/loss across all closed trades
- **Win Rate** — % of closed trades that resolved profitably
- **Open Positions** — Live table of all active bets with entry prices
- **Trade History** — Last 20 trades with Claude's reasoning for each
- **Compounding Projection Chart** — Projected 12-month bankroll at
  conservative (5%/mo), moderate (10%/mo), and aggressive (20%/mo) returns

## Bankroll Compounding Strategy

The bot follows a **balanced fractional Kelly** approach:

| Parameter | Value |
|-----------|-------|
| Starting bankroll | $100 |
| Max single trade | 10% of current bankroll |
| Minimum edge to trade | 5% |
| Sizing formula | `edge × confidence × 0.25` (capped at 10%) |

The compounding happens naturally: winning trades increase the bankroll, which
means the next trade's dollar amount is slightly larger. Losing runs shrink
bets automatically — you can never lose more than 10% per trade.

## Swapping in Real Trade Execution

When you're ready to trade with real money, replace the simulated execution
in `executor.py` with real Polymarket API calls.

### 1. Install the Polymarket Python client

```bash
pip install py-clob-client
```

### 2. Replace `execute_trade()` in `executor.py`

Find the bankroll deduction section and replace it with a real order:

```python
from py_clob_client.client import ClobClient

clob = ClobClient(
    host="https://clob.polymarket.com",
    chain_id=137,   # Polygon
    private_key=os.environ["POLYMARKET_PRIVATE_KEY"],
    signature_type=2,
    funder=os.environ["POLYMARKET_FUNDER_ADDRESS"]
)

order = clob.create_and_post_order(OrderArgs(
    price=entry_price,
    size=bet_size,
    side=BUY if recommendation == "YES" else SELL,
    token_id=trade["market_id"],  # live markets carry the real token ID in market_id
))
trade["order_id"] = order.orderID
```


## Adjusting the Strategy

| Setting | Location | Default |
|---------|----------|---------|
| Starting bankroll | `executor.py` → `INITIAL_BANKROLL` | $100 |
| Max bet fraction | `executor.py` → `MAX_BET_FRACTION` | 0.10 (10%) |
| Minimum edge to trade | `bot.py` → `MIN_EDGE` | 0.05 (5%) |
| Claude model | `bot.py` → `MODEL` | `claude-sonnet-4-6` |
| Use live markets | `.env` → `USE_LIVE_MARKETS` | `false` |
| Min market liquidity | `market_fetcher.py` → `MIN_LIQUIDITY` | $1 000 |
| Min 24h volume | `market_fetcher.py` → `MIN_VOLUME_24H` | $1 000 |
| Max markets fetched | `market_fetcher.py` → `MAX_MARKETS` | 20 |

## Common Issues

**`ANTHROPIC_API_KEY` not found**
- Copy `.env.example` to `.env` and add your key, then re-run.

**`dashboard_data.js` not found**
- Run `python bot.py` at least once — the file is auto-generated.
  An initial empty-state file is included so the dashboard renders before the first run.

**All markets show as SKIP**
- Claude only trades when it finds a genuine edge > 5%. This is intentional.
  Try adjusting `MIN_EDGE` to `0.03` (3%) in `bot.py` or switch to live markets
  which have a wider variety of current prices.

**Live market fetch fails**
- Check your internet connection and that `USE_LIVE_MARKETS=true` is in `.env`.
- The bot automatically falls back to `markets_cache.json` (if it exists) or
  `mock_markets.json`, so it will still run.

**Dashboard not auto-refreshing on phone**
- Access it via the HTTP server (`python -m http.server 8080`),
  not by opening the file directly on your phone.

**Vercel deployment shows stale data**
- Run `vercel --prod` after each `python bot.py` run, or set up GitHub Actions
  to auto-deploy when `dashboard_data.js` is committed (see Vercel section above).
